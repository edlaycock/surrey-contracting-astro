Getting Started

Logging in, navigating the interface, and understanding how publishing works

1.1 Logging In

  1. Open your browser and go to https://surreycontracting.sanity.studio
  2. Click Log in and sign in with the email address you were invited with.
  3. You will land on the Desk — the main editing interface showing the content sidebar on the left.
📸
Screenshot: Sanity Studio login screen
Show the surreycontracting.sanity.studio login page with the "Log in" button visible
ℹ️
First time? Check your email for an invitation from Sanity (sender: no-reply@sanity.io). Click Accept invitation before trying to log in. If you don't see it, check your spam folder.
📸
Screenshot: Sanity Studio Desk (logged in)
Show the full Desk view after logging in — sidebar on the left, content list in the centre, with the "Project / Case study" section visible

1.2 The Sidebar

The left sidebar shows all content sections you can manage. Currently the site has one content type:

Sidebar item What it contains Where it appears on the website
Project / Case study All project pages and case studies /projects and individual /projects/[name] pages
💡
Tip: Click the section name in the sidebar to see a list of all documents of that type. Click any document in the list to open and edit it.

1.3 Drafts vs Published

Sanity uses a draft / publish workflow. This means you can work on changes privately before they go live on the website.

State What it means Visible on website?
Draft Changes saved in Sanity but not yet published No
Published The version visitors see on surreycontracting.co.uk Yes
ℹ️
Key point: The live website always shows the last published version. Editing a document does not change the live site until you click Publish.

Projects / Case Studies

Creating, editing, and managing project pages on the website

Each project in Sanity becomes a dedicated page at surreycontracting.co.uk/projects/[slug], and also appears as a card on the main projects listing page at /projects.

📸
Screenshot: Projects list in Sanity
Show the "Project / Case study" section selected in the sidebar, with the list of existing projects visible in the centre panel

2.1 Creating a New Project

  1. In the left sidebar, click Project / Case study.
  2. Click the compose / pencil icon at the top right of the list panel, or click + New document.
  3. A blank document opens on the right. Work through each field from top to bottom — see the field reference in section 2.2.
  4. When you are ready for the project to appear on the website, click the blue Publish button (top right). The site updates within a few minutes.
📸
Screenshot: New blank project document
Show a freshly created project document with all empty fields visible — highlight the compose icon used to create it
💡
Tip: You don't need to fill every field before saving. Sanity saves automatically as you type. Come back and complete it before publishing.

2.2 Field Reference

Basic Information

Field Required? Description Example
Title Required The project name. Shown as the page heading, card title, and in Google search results. Tarmac Driveway, Fetcham, Surrey
Slug (URL) Required The web address. Click Generate to create from the Title. Use lowercase letters, numbers, and hyphens only — no spaces. tarmac-driveway-fetcham
Summary Required A 1–2 sentence description. Used on cards, the hero lede, and Google search snippets. Be specific: include service type, location, and a key fact. 100sqm tarmac driveway in Fetcham, Surrey, with base course and SMA wearing course.
Hero Image Required The large banner image for the project page and card photo. Best size: landscape, 1600×900px or larger. Always fill in the Alt text after uploading. —

Categorisation

Field Required? Description Options
Categories Required Service categories for this project. Used for the filter buttons on the projects page. Select all that genuinely apply. Drainage · Driveways · Earthworks · Groundworks · Hard Landscaping · Surfacing
Sector Optional The market sector. Shown as a small tag on the project hero image. Residential · Commercial · Education · Commercial / Public Realm · Public Sector

Project Info Card

These fields populate the "Project info" sidebar shown on each project page alongside the body text.

Field Required? Description Example
Client Optional The client name as shown in the project info card. Private residential or Sutton High School
Location Optional Where the project took place. Fetcham, Surrey
Services Optional A brief label for the services used. Separate multiple services with a middle dot · Tarmac · Drainage
Year Optional Year the project was completed. Hidden if Duration is set. 2024
Duration Optional Used instead of Year for longer projects with a notable programme length. 14 weeks
Status Optional Shown as a badge in the info card. Defaults to "Completed". Completed

Field Required? Description

2.3 Editing an Existing Project

  1. Click Project / Case study in the sidebar.
  2. Click the project you want to edit from the list.
  3. Make your changes — Sanity saves automatically as you work.
  4. Click Publish to push changes live. Until you publish, the live website shows the previous published version.
ℹ️
Editing a live project is safe. Your changes are saved as a draft and the live website is unchanged until you click Publish. You can take as long as you need.

2.4 Warnings

⚠️
Do not change the Slug of a project that is already live on the website. Changing the slug changes the URL, which breaks any existing links or bookmarks to that page. If you must rename a project, contact your web developer to set up a redirect first.
⚠️
Hero Image and Categories are required. You will not be able to publish a project without them. If the Publish button doesn't work, check these fields are filled in.
⚠️
Deleting a project is permanent. Deleting removes the page from the live website immediately. There is no recycle bin. If you want to temporarily hide a project, use Unpublish instead (see Section 3.4).

Publishing, Drafts & Unpublishing

Controlling what appears on the live website

3.1 Publishing a Document

  1. Open the document you want to publish.
  2. Check your content looks correct.
  3. Click the blue Publish button in the top right corner.
  4. The document is now live. The website updates within a few minutes.
📸
Screenshot: The Publish button
Show the top-right corner of a Sanity document with the blue "Publish" button clearly visible, and the dropdown arrow next to it

3.2 Saving a Draft Without Publishing

Sanity saves your work automatically as you type. Simply close the document or navigate away and your draft is stored safely. The live website is unchanged.

You can also click the dropdown arrow next to the Publish button and choose Save as draft to explicitly confirm the save without publishing.

💡
Tip: Documents with unsaved/unpublished changes show an "Edited" indicator. Documents that are published and up to date show "Published".

3.3 Editing a Published Document

You can edit a published document at any time without taking it offline. Sanity creates a new draft automatically — the live website continues to show the last published version until you click Publish again.

  1. Open the published document.
  2. Make your changes — an "Edited" badge appears showing there are unpublished changes.
  3. When ready, click Publish to update the live site.

3.4 Unpublishing a Document

Unpublishing removes a document from the live website but keeps it in Sanity so you can re-publish it later. Use this instead of deleting when you want to temporarily hide content.

  1. Open the document you want to hide.
  2. Click the dropdown arrow (▾) next to the Publish button.
  3. Choose Unpublish.
  4. The document is removed from the live site immediately but remains in Sanity as a draft.
📸
Screenshot: Publish button dropdown menu
Show the dropdown menu opened from the arrow next to the Publish button, with "Unpublish" and "Discard changes" options visible

3.5 Discarding Draft Changes

If you've made changes you don't want to keep, you can revert to the last published version.

  1. Click the dropdown arrow (▾) next to the Publish button.
  2. Choose Discard changes.
  3. The draft is deleted and the document reverts to its last published state.
⚠️
This cannot be undone. Discarding changes permanently removes your draft edits. Make sure you don't need them before confirming.

Writing Style Guide

Brand rules for consistent, professional content across the website

4.1 Tone

DoDon't
Professional, clear, and straightforward Over-formal or overly casual
Use technical terms where relevant — the audience understands tarmac, macadam, SMA Avoid jargon with no explanation when writing for a broad audience
Third person for project descriptions: "Surrey Contracting carried out..." First person: "We carried out..."

4.2 Capitalisation

RuleExample
Always write the full company name — never abbreviate Surrey Contracting — not SC or Surrey C
Capitalise proper nouns: client names, place names, product names Premier Inn, Sutton High School, Virginia Water
Service types are lowercase unless starting a sentence tarmac, drainage, earthworks

4.3 Numbers & Measurements

RuleExample
Use numerals for quantities 120 tonnes, 30sqm, 14 weeks
Spell out numbers at the start of a sentence One hundred and twenty tonnes of material was removed...
Use a middle dot (·) to separate services in the Services field Tarmac · Drainage

4.4 Summaries

The Summary field appears on project cards, the project page hero, and Google search snippets. It should be concise but specific.

RuleExample
Maximum 1–2 sentences —
Include: service type + location + one key fact Emergency tarmac footpath repairs at Sutton High School, completed over a weekend to minimise disruption.
Mention size or scale where relevant 100sqm tarmac driveway installation with base course and SMA wearing course in Fetcham, Surrey.

4.5 Images

RuleDetail
Always fill in Alt text Describe the photo factually. Completed tarmac driveway with granite sett edging, West Humble — not photo1
Landscape orientation only Portrait photos will be cropped unexpectedly. Aim for 16:9 or 4:3 ratio.
Minimum size for hero image 1600 × 900 px
Prefer finished work for hero images "After" photos make better hero images than mid-construction shots where possible.

4.6 Categories

RuleDetail
Choose the most specific category If primarily a driveway, select Driveways even if surfacing was involved.
Use multiple categories sparingly Only if the project genuinely covered both service areas substantially — not just because a service was mentioned.
Categories drive the filter buttons The filters on the /projects page are built directly from whatever categories your projects use. Keep them consistent.
